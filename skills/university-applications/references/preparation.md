# 占卜准备指南

---

## 通用原则

### 任何解读之前
1. **明确目的**：有一个清晰的问题或关注领域
2. **保持开放**：不带刚性预期
3. **记录时间**：记下解读的确切日期/时间
4. **环境整洁**：选择安静、舒适的空间

### 心理准备
1. 静心：深呼吸几次
2. 专注：集中于你的问题
3. 诚心：以真诚的意图对待
4. 接受：对意外答案保持开放

---

## 各方法所需信息速查

> ### ⚠️ 隐私与最小化提醒（请先阅读）
>
> 下表列出的「必需信息」均属于**敏感个人数据**：包括出生年月日时、出生地/地址、姓名、双手或正面照片、家庭成员（如配偶/父母/子女）出生信息等。
> - **最小化原则**：只在你**真正想做对应解读**时才提供对应字段；不打算做该体系解读时**完全不需要**填写。
> - **本地存储**：本 skill 的所有用户档案仅写入本机 `data/profiles/<userId>.json`，不向任何外部服务上传，详见 [SKILL.md 数据留存与用户控制](../SKILL.md#-数据留存与用户控制隐私)。
> - **保留期**：保存到你主动 `node scripts/profile.js delete <userId>` 为止。该脚本默认采用**软删除（可 30 天内手动恢复）**，加 `--purge` 才会物理删除。
> - **照片/平面图等大体积数据**：本 skill 不会自动持久化为档案字段；请仅在当次会话中粘贴使用，会话结束后由你自行决定是否保存。
> - **未成年人**：尽量不要为未成年人长期保留档案；模板严禁对未成年人下宿命标签。
> - **家庭成员**：仅在确实需要合婚/亲子合盘等多体系交叉验证时才录入；任何时刻可用 `profile.js delete` / 直接编辑 JSON 删除。
> - **日志最小化**：`profile.js show / list` 默认脱敏；如需完整字段请用 `--full`，并避免在共享终端 / CI / cron 邮件中执行。
> - **不要把以下信息当作专业凭证**：不要据此作出医疗、法律、财务、婚姻、就业等重大决策。

| 方法 | 必需信息 | 可选信息 |
|------|---------|---------|
| 易经 / 六爻 | 明确问题 | 起卦时间 |
| 八字 | 出生年月日时 + 地点 | 性别 |
| 紫微斗数 | 出生年月日时 + 地点 | 性别 |
| 塔罗 | 一个清晰的问题 | 牌组偏好 |
| 西方星盘 | 出生日期 + 时间（精确到分钟）+ 地点 | — |
| 数字命理 | 出生日期 + 姓名 | — |
| 手相 | 双手清晰照片 | 年龄、性别 |
| 面相 | 正面清晰照片 | 年龄 |
| 风水 | 平面图/地址 + 朝向 | 居住者出生日期 |
| 奇门遁甲 | 排盘时间 + 问题 | 问事类型 |
| 梅花易数 | 问题 + 数字/时间/触发物 | — |

## 提问指南

### ❌ 错误方式
- 问模糊问题（"我会怎样？"）
- 一次问多个问题
- 期待绝对答案
- 忽略矛盾信息

### ✅ 正确方式
- 一次一个清晰问题
- "我应该……？" 或 "关于……我需要知道什么？"
- 接受指引而非预言
- 用解读获取洞察而非借口

---

## 各类问题示例

### 感情/关系
- "关于我目前的关系，我需要知道什么？"
- "我应该追求这段连接吗？"

### 事业/工作
- "什么职业路径适合我？"
- "什么时候适合求职？"

### 决策
- "在这个决定中我应该考虑什么？"
- "如果我选择 X，可能的结果是什么？"

### 通用指引
- "我现在需要知道什么？"
- "这个月/年我应该关注什么？"

---

## 重要提醒

1. **不是预言**：占卜显示趋势，不是固定未来
2. **可变**：能量会变化，结果也会
3. **配合**：解读与自由意志配合，而非对抗
4. **参考**：用指引做出明智选择

---

## 专业读者的红旗

| ✅ 好的迹象 | ❌ 危险信号 |
|------------|-----------|
| 清楚解释方法 | 保证特定结果 |
| 合理定价 | 推销昂贵套餐 |
| 愿意解释过程 | 拒绝解释方法 |
| 不做保证 | 声称超自然力量 |
| 有口碑/评价 | 索要过多个人信息 |
