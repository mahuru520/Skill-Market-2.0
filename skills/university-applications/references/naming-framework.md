# 起名 / 命名解读框架 (Naming Framework)

> ⚠️ **文化伦理提示**
> - 起名是伴随一生的文化标识，影响深远。
> - 本 skill 仅提供基于传统命理文化的分析参考，**不替代专业起名服务**。
> - 父母/本人对名字的最终决定权高于一切命理分析。
> - 不保证名字"改运"效果，不承诺任何具体人生结果。

---

## 适用场景

| 场景 | 示例 |
|------|------|
| 新生儿起名 | "给孩子起个名字" |

| 笔名/艺名/网名 | "想取个笔名" |

## 资料完整度分级

| 级别 | 条件 | 处理方式 |
|------|------|---------|
| **S 级** | 完整出生八字 + 性别 + 姓氏 + 家族辈分字 + 父母期望 | 八字用神分析 + 五行补益 + 音形义全审 |
| **A 级** | 出生年月日（有时辰）+ 性别 + 姓氏 | 五行偏弱分析 + 选字方向 |
| **B 级** | 出生年月日（无时辰）+ 姓氏 | 粗略五行判断 + 通用吉字推荐 |
| **C 级** | 仅有姓氏 + 性别 | 通用音律+寓意方向，不做五行断言 |

## 核心框架

### 1. 八字用神分析（S/A 级时）

基于出生时间排八字，分析：
- **日主强弱**：判断命主自身能量状态
- **五行分布**：金木水火土各自占比
- **喜用神**：最需要的五行能量
- **忌神**：需要避开的五行能量

**选字原则**：
- 名字用字偏旁/字义尽量补益喜用神五行
- 避免名字用字与忌神五行冲突
- 不追求五行"齐全"，而追求"平衡"与"流通"

| 五行 | 常见偏旁/部首 | 字义倾向 |
|------|-------------|---------|
| 金 | 钅、金、刂、刀、玉 | 刚、锐、清、明、义 |
| 木 | 木、艹、禾、竹 | 生、长、仁、和、柔 |
| 水 | 氵、水、冫、雨 | 智、润、慧、流、清 |
| 火 | 火、灬、日、光 | 礼、热、明、进、辉 |
| 土 | 土、山、石、田、阜 | 信、厚、稳、实、容 |

### 2. 字音审查

#### 声韵搭配
- **避免拗口**：声母、韵母不宜连续重复或冲突
- **平仄协调**：姓与名的声调搭配宜有起伏
  - 平声（第一、二声）+ 仄声（第三、四声）交替为佳
  - 三平或三仄连续读来易显单调

#### 谐音排查
- **必须排查**名字整体是否有不雅谐音
- 注意方言谐音（如粤语、闽南语等）
- 注意连读变调后的谐音

### 3. 字形审查

- **结构均衡**：左右结构、上下结构搭配协调
- **笔画适中**：不宜过繁（书写困难）或过简（显得单薄）
- **避免生僻**：优先常用字，避免生僻字导致生活不便
- **风格适配**：根据用户明确偏好的风格方向（如儒雅、刚健、柔和、中性、现代、古典等）提供建议；不预设性别化审美，尊重用户自我认同。

### 4. 字义审查

- **本义**：字的本源含义
- **引申义**：文化语境中的联想
- **时代感**：避免过于老气或过于跳脱
- **正向寓意**：积极、美好、有文化内涵

### 5. 三才五格（参考性）

天格（姓氏笔画+1）· 人格（姓末字+名首字）· 地格（名字笔画和）· 外格 · 总格

> 三才五格为近代日本传回的方法，**仅作文化参考**，不做绝对评判标准。

## 起名输出结构

### 标准版（S/A 级）

1. **八字五行简析**：日主强弱、五行分布、喜用神方向
2. **起名方向**：五行补益方向 + 音律偏好
3. **推荐名字**：3-5 个，每个附
   - 读音、字形结构
   - 字义解析
   - 五行属性
   - 音形义综合评价
4. **备选方向**：如果用户有特定偏好（如家谱辈分字），提供适配方案
5. **避用提醒**：谐音排查结果、忌用字方向
6. **自我决定提示**：强调最终决定权在家长/本人
7. **点醒句**：收尾

### 轻量版（B/C 级或仅问"这个名字如何"时）

1. **名字简析**：音、形、义快速评估
2. **五行倾向**：粗略判断（如可判断时）
3. **改善建议**：如有明显问题，给调整方向
4. **一句总结**：是否推荐及理由

## 硬性边界

| 禁止行为 | 原因 |
|---------|------|
| 声称名字可以"改命""转运" | 禁止夸大与商业欺诈 |
| 断言某个名字必定导致好坏 | 禁止宿命论 |
| 替代父母/本人做最终决定 | 尊重命名自主权 |
| 使用极端生僻字或自造字 | 避免给用户生活带来实际困扰 |
| 推销付费起名服务 | 禁止商业引导 |
| 对已有名字下定性负面标签 | 避免伤害用户情感 |
