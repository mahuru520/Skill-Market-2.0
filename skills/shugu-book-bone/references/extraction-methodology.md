# 书籍思维框架提炼方法论

> 从一本书的原始内容到可运行的心智模型和方法论

## 一、心智模型识别的三重验证（书籍版）

一个论点要被认定为「心智模型」而非「书中的一页内容」，必须通过三重验证：

### 验证1: 章间复现

同一个思维框架在书中至少2个不同章节/场景中出现。

**不是**简单重复同一个观点——而是用同一个思维工具分析不同问题。

例：《原子习惯》的「身份认同驱动行为」——
- 在第2章（习惯与身份的关系）提出核心概念
- 在第3章（4步习惯循环）用身份认同解释每个步骤的心理机制
- 在第12章（习惯与人际关系）用身份认同解释社会行为的传染
- 在第16章（习惯与组织文化）用身份认同解释团队行为
→ 跨4个章节复现 → 这是真正的心智模型

例：《思考，快与慢》的「系统1/系统2」——
- 在第1部分（两种系统）建立基础框架
- 在第2部分（启发法与偏误）用系统1解释判断错误
- 在第3部分（过度自信）用系统1/2的冲突解释认知错觉
- 在第4部分（选择与风险）用系统1解释前景理论
- 在第5部分（两个自我）用系统1/2解释幸福与记忆
→ 贯穿全书5个部分 → 这是真正的心智模型

**如果一个观点只在某一章出现一次** → 降级为「方法论步骤」或「案例」，不认定为心智模型。

### 验证2: 有生成力

用这个框架可以推断作者对书中未讨论问题的可能立场。

例：如果「损失厌恶」是心智模型——
- 面对新的金融产品定价 → 作者会先问「用户的损失感知是什么」
- 面对政策设计 → 作者会关注「政策让人们感到失去了什么」
→ 能生成新推断 → 这是真正的心智模型

### 验证3: 有排他性

这个视角不是同类书的标配。

例：「身份认同驱动行为改变」在《原子习惯》中是独特的——
- 多数习惯类书籍只谈「奖励-行为」循环（触发→行动→奖励）
- James Clear 独特地加入了「身份」层：行为改变的最终形态不是做什么，而是成为谁
→ 有区分度 → 这是值得提炼的心智模型

**通过情况判定**：
- 三重都通过 → 心智模型
- 仅1-2重通过 → 降级为「方法论步骤」或「案例模式」
- 0重通过 → 丢弃

## 二、方法论提取标准

从书中提取方法论时，必须满足三个「可」：

### 可执行

能转化为具体的步骤或检查清单，不只是抽象原则。

| 可执行 | 不可执行 |
|--------|---------|
| 「先识别触发点，再设计替代行为」 | 「坚持做正确的事」 |
| 「列出3个最坏结果，逐一评估概率」 | 「保持理性思考」 |

### 可验证

有书中案例或实验数据支撑。

- 作者用什么实验/案例证明这个方法有效？
- 数据来自作者自己的研究还是引用他人？
- 样本量和适用范围是什么？

### 可迁移

能应用于书中未提及的新场景。

- 这个方法只在书里的特定场景有效，还是具有通用性？
- 如果场景参数变化（行业、文化、时间），方法是否仍然适用？

**排除标准**：
- 鸡汤式建议（「坚持就好」「相信自己」「不要放弃」）
- 只适用于极窄场景无法迁移
- 书中只提了一次且没有展开论证

## 三、概念/词汇表提取标准

提取作者在书中使用的关键术语。

### 纳入标准（满足任一）

- **自创术语**：作者创造的新词（如「反脆弱」「系统1」「原子习惯」）
- **重新定义**：对常见术语赋予了新的含义或范围
- **结构性重要**：在书中多次出现，且承载论证的关键术语

### 排除标准

- 书中顺带提及但未展开的术语
- 行业标准术语且作者没有重新解释
- 只在某一段落出现一次的术语

### 记录格式

| 字段 | 说明 |
|------|------|
| 概念名 | 术语原文（中英文如有） |
| 定义 | 一句话定义 |
| 出现章节 | 首次出现 + 核心讨论章节 |
| 关联概念 | 与书中其他概念的关系 |

## 四、案例模式提取

书中案例很多，但不是每个都有价值。提取标准：

### 反复出现

作者用这个类型的案例论证多个观点。

### 揭示框架

案例本身展示了某个心智模型的运作。

### 可迁移

案例模式可以映射到其他领域。

例：《思考，快与慢》中「专家预测 vs 统计模型」的案例模式——
- 用多个领域（投资、医学、政治）的对比案例论证同一框架
- 揭示的心智模型：人类直觉在复杂预测任务中系统性劣于简单算法
- 可迁移到任何依赖专家判断的决策场景

## 五、矛盾处理原则

矛盾是书的核心特征，不是需要修复的Bug。

### 三种矛盾类型

1. **时间性矛盾**（观点演化）
   - 作者在本书不同章节或不同版本中表达了不同观点
   - 处理：记录演化轨迹，标注「早期」「近期」

2. **领域性矛盾**（不同场景不同规则）
   - 作者在某个场景主张X，在另一个场景主张Y
   - 处理：分领域记录，不强求统一

3. **本质性张力**（价值观/框架内在冲突）
   - 例：某书既强调「数据驱动决策」又强调「相信直觉」
   - 处理：明确记录为「核心张力」

### 错误的处理方式
- 选一边忽略另一边
- 编一个调和的解释
- 假装矛盾不存在

## 六、书籍类型的差异处理

不同类型的书，提取的重点和数量预期不同：

### 哲学/理论型（如《思考，快与慢》《反脆弱》）

| 维度 | 数量 | 重点 |
|------|------|------|
| 心智模型 | 5-7个 | 作者的世界观和认知框架 |
| 方法论 | 1-3个 | 理论如何转化为行动 |
| 概念表 | 10-15个 | 作者自创术语体系 |
| 案例模式 | 3-5个 | 作者如何论证 |
| 表达风格 | 提取 | 学术叙事方式 |

### 方法论/实用型（如《原子习惯》《精益创业》）

| 维度 | 数量 | 重点 |
|------|------|------|
| 心智模型 | 3-5个 | 方法论背后的假设 |
| 方法论 | 5-8个 | 具体操作步骤和检查清单 |
| 概念表 | 5-8个 | 方法论专用术语 |
| 案例模式 | 2-3个 | 方法论的典型应用 |
| 表达风格 | 可选 | 通常不重要 |

### 叙事/传记/合集型（如《穷查理宝典》《人类简史》）

| 维度 | 数量 | 重点 |
|------|------|------|
| 心智模型 | 3-5个 | 贯穿叙事的核心框架 |
| 方法论 | 3-5个 | 从叙事中可提取的行动原则 |
| 概念表 | 5-10个 | 核心术语和概念 |
| 案例模式 | 3-5个 | 叙事/论证模式 |
| 表达风格 | 提取 | 叙事风格是重要特征 |

### 教科书/技术型

| 维度 | 数量 | 重点 |
|------|------|------|
| 心智模型 | 2-4个 | 学科的核心思维方式 |
| 方法论 | 5-8个 | 技术步骤、最佳实践 |
| 概念表 | 10-20个 | 术语体系 |
| 案例模式 | 1-2个 | 典型应用场景 |
| 表达风格 | 跳过 | 技术书无风格价值 |

## 七、质量自检清单

生成Skill后，用以下问题自检：

### 心智模型
- [ ] 每个模型都有至少2个不同章节的证据？
- [ ] 模型数量在3-7个之间？
- [ ] 每个模型都有明确的应用场景和局限？
- [ ] 模型之间有张力但不矛盾？

### 方法论
- [ ] 每条规则都有具体案例支撑？
- [ ] 可以被新情况触发（不只适用于书中案例）？
- [ ] 步骤足够具体，可以当作检查清单使用？

### 概念词汇表
- [ ] 每个概念都有清晰定义？
- [ ] 标注了出现的章节？
- [ ] 说明了与其他概念的关系？

### 应用边界
- [ ] 明确写了适合和不适合的场景？
- [ ] 标注了作者的已知盲区？

### 整体
- [ ] 用这本书的眼睛看一个新问题，能得到有价值的视角？
- [ ] 不是书中内容的摘要拼凑，而是框架的运行？
- [ ] 删掉书名后，还能认出这是哪本书的思维方式？
