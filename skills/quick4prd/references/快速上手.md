# 智能体需求文档生成 Skill：快速上手

适合第一次导入 skill 后，想尽快跑通一遍的人。

## 0. 先装依赖

进入 skill 的 `scripts/` 目录执行一次：

```bash
cd ~/.workbuddy/skills/智能体需求文档生成skill/scripts
npm install
```

只需要首次执行一次。

## 1. 先判断你属于哪种场景

| 你的目标 | 推荐入口 |
| --- | --- |
| 我要先新建一份源文档，再顺手生成首版快照 | `gen_template.js` |
| 源 Markdown 已经写好了，只想生成新版本快照 | `gen_from_md.js --source` |
| 我希望把固定参数存起来反复使用 | `--profile` |
| 我要一次生成多份文档 | `--manifest` |
| 我要批量生成，而且多个文档共享同一输出目录 | `--manifest --profile` |

## 2. 三个最常用命令

### 场景 A：新建首版文档

```bash
/usr/local/bin/node ~/.workbuddy/skills/智能体需求文档生成skill/scripts/gen_template.js \
  --output-dir /absolute/path/to/doc \
  --basename 示例项目-用户管理功能设计 \
  --title 用户管理功能设计 \
  --system 示例管理平台 \
  --project 示例项目
```

### 场景 B：生成单文档快照

```bash
/usr/local/bin/node ~/.workbuddy/skills/智能体需求文档生成skill/scripts/gen_from_md.js \
  --source /absolute/path/to/doc/示例项目-用户管理功能设计.md \
  --output-dir /absolute/path/to/doc
```

### 场景 C：批量生成

```bash
/usr/local/bin/node ~/.workbuddy/skills/智能体需求文档生成skill/scripts/gen_from_md.js \
  --manifest ~/.workbuddy/skills/智能体需求文档生成skill/assets/examples/feature-design.manifest.json
```

## 3. 推荐上手路径

### 路径 1：最快验证 skill 是否可用

1. 复制 `assets/examples/示例功能设计.md` 到你的项目文档目录
2. 修改文档标题、版本号、修订记录
3. 执行单文档生成命令
4. 检查是否产出：
   - `xxx_v0.x.md`
   - `xxx_v0.x.docx`
   - `backup/`（如目录中已有旧快照）

### 路径 2：给项目建立长期可复用配置

1. 复制 `assets/examples/feature-design.profile.json`
2. 改成你项目自己的输出目录、项目名、系统名、模块路径
3. 后续用 `--profile` 初始化新文档
4. 如需批量生成，再补一份 manifest

## 4. 生成完成后检查什么

- 源文件仍然保留无版本号文件名
- 快照文件名带版本号，例如 `_v0.3`
- `.md` 与 `.docx` 快照版本一致
- 文档内“文档版本”与“修订记录”一致
- 旧快照已移动到 `backup/`

## 5. 等保合规引用说明

若你的功能模块涉及账号认证、权限控制、操作日志、敏感数据、导入导出或接口安全等要求：

1. **参考基准文件**：`references/通用等保三级安全要求.md` 已覆盖身份鉴别、访问控制、操作日志、敏感数据保护、文件操作安全、数据传输安全六大类通用规范
2. **按需引用**：在功能设计文档的 `10.1 合规与安全要求` 中，仅列出本模块实际涉及的章节，并补充模块特有说明
3. **引用格式**：`本模块[功能点]要求参见《通用等保三级安全要求》§[章节号]`
4. **项目定制**：若具体项目需要调整合规口径，直接修改 `references/通用等保三级安全要求.md`，后续文档自动按更新后的内容引用

## 6. 常见建议

- 公共配置放 profile，单文档差异放 manifest
- 项目专属业务规则应写进项目自己的源文档或项目模板中
- 通用合规基线（如等保要求）统一维护在 skill 的 `references/` 下，功能文档按需引用
- 如果章节结构和通用模板差异很大，优先自建模板，再通过 `--template` 指定
