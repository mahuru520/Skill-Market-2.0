## 一、身份鉴别

### 1.1 登录认证

| 规则项   | 规则说明                                      |
| ----- | ----------------------------------------- |
| 认证方式  | 密码 + 图形验证码双重校验，缺一不可                       |
| 图形验证码 | 每次进入登录页自动刷新；点击验证码图片可手动刷新；验证码有效期60秒，超时自动刷新 |
| 密码输入  | 密码框默认隐藏明文，提供"显示/隐藏"切换图标                   |

### 1.2 密码策略

| 规则项    | 规则说明                                                          |
| ------ | ------------------------------------------------------------- |
| 密码长度   | 8\~20位                                                        |
| 复杂度要求  | 必须同时包含以下至少三类字符：大写字母（A-Z）、小写字母（a-z）、数字（0-9）、特殊字符（`!@#$%^&*_-`） |
| 禁止规则   | 不能包含用户名；不能为纯连续数字（如 `12345678`）；不能为键盘连续字符（如 `qwerty`）          |
| 有效期    | 90天，到期强制提醒修改                                                  |
| 到期提醒   | 到期前7天，每次登录时顶部展示提醒条："您的密码将于X天后到期，请及时修改"                        |
| 到期强制修改 | 密码到期后，登录时强制跳转至修改密码页，未修改不可进入系统                                 |
| 历史密码限制 | 新密码不能与当前密码相同                                                  |
| 密码存储要求 | 推荐使用国密算法（如SM3）进行存储                                          |

### 1.3 账号锁定与解锁

| 规则项    | 规则说明                              |
| ------ | --------------------------------- |
| 触发条件   | 同一账号连续登录失败5次，账号自动锁定               |
| 锁定效果   | 账号锁定后无法继续登录，登录页提示"账号已锁定，请联系管理员解锁" |
| 解锁方式   | 支持自动解锁（如3分钟后）或管理员手动解锁；具体策略根据项目需求配置 |
| 失败计数重置 | 登录成功后，失败计数清零                      |

<br />

### 1.4 会话管理

| 规则项  | 规则说明                         |
| ---- | ---------------------------- |
| 超时时长 | 无操作30分钟后，系统自动退出登录（时长可根据项目调整） |
| 超时处理 | 自动退出后跳转至登录页，并提示"登录已超时，请重新登录" |

### 1.5 密码重置规则

| 规则项  | 规则说明                                    |
| ---- | --------------------------------------- |
| 操作权限 | 授权角色可执行重置密码操作                           |
| 强制修改 | 账号被重置密码后，用户下次登录时系统强制要求修改密码，不输入新密码无法进入系统 |

## 二、访问控制

### 2.1 RBAC 最小权限原则

| 规则项  | 规则说明                                    |
| ---- | --------------------------------------- |
| 权限模型 | 基于角色的访问控制（RBAC），每个用户通过角色获得对应的功能权限       |
| 最小权限 | 每个角色只赋予完成本职工作所需的最小权限集，禁止超范围授权           |
| 默认拒绝 | 未被明确授权的功能入口，前端不展示菜单和按钮；后端接口同步鉴权拦截，返回403 |
| 角色分配 | 用户角色由系统管理员分配，不允许用户自行修改角色                  |

### 2.2 数据权限隔离

| 规则项    | 规则说明（根据项目实际调整）                     |
| ------ | --------------------------------- |
| 系统管理员 | 可查看全量数据，不受数据范围限制                   |
| 普通用户   | 只能查看和操作权限范围内的数据，不可越权访问            |
| 访客     | 仅可访问公开页面，不可访问任何受保护的后台数据           |

### 2.3 高危操作二次确认

以下操作在前端须弹出确认弹窗，用户主动确认后方可执行：

| 操作类型 | 确认文案示例                  |
| ---- | ----------------------- |
| 删除 | 是否确认删除该记录？删除后不可恢复。      |
| 停用账号 | 是否确认停用该账号？停用后该用户将无法登录。  |
| 重置密码 | 是否确认重置该用户密码？重置后初始密码将生效。 |

> **说明**：具体项目可根据业务需要决定是否提供批量删除功能。

## 三、操作日志规范

### 3.1 记录范围

系统中所有操作均会记录到操作日志。

### 3.2 日志字段规范

日志粒度为**操作级**，记录每次操作的接口调用的详情。

| 字段   | 说明                  | 示例                  |
| ---- | ------------------- | ------------------- |
| 登录信息 | 执行操作的用户名 + 姓名       | admin\_001（张三）      |
| 操作时间 | YYYY-MM-DD HH:mm:ss | 2026-03-31 09:00:00 |
| 操作模块 | 所属功能模块名称以及操作名字      | 用户信息管理/修改           |
| 请求地址 | 请求接口的地址信息           | /api/user/update    |
| 请求参数 | 入参                  | {"id":"2"}        |
| 请求方式 | 请求的是 post/get       | post                |
| 操作IP | 操作发起的客户端IP地址        | 192.168.1.100       |
| 返回参数 | 返回的结果               | {"code":200}        |
| 操作状态 | 成功 / 失败             | 成功                  |
| 消耗时间 | 请求接口消耗的时间           | 77 毫秒               |

<br />

### 3.3 日志存储与查询

| 规则项  | 规则说明                           |
| ---- | ------------------------------ |
| 不可篡改 | 日志只允许查看，不允许修改或删除，接口层强制只读       |
| 保留时长 | 日志保留时长不少于 **6个月**              |
| 查询入口 | 系统管理 → 操作日志（或等效入口）              |
| 查询权限 | 系统管理员可查看全量日志；其他角色根据权限配置         |

## 四、敏感数据脱敏规范

### 4.1 脱敏规则

| 字段类型      | 脱敏规则                     | 展示示例                 |
| --------- | ------------------------ | -------------------- |
| 手机号（11位）  | 保留前3位和后4位，中间4位替换为 `*`    | `138****1234`        |
| 身份证号（18位） | 保留前6位和后4位，中间8位替换为 `*`    | `110101********1234` |
| 邮箱地址      | `@` 前保留首字符，其余替换为 `*`     | `z***@example.com`   |
| 银行卡号      | 保留前4位和后4位，中间替换为 `*`      | `6222****8888`       |

### 4.2 脱敏适用范围

| 展示场景   | 是否脱敏   |
| ------ | ------ |
| 列表页展示  | 是，统一脱敏 |
| 查看详情弹窗 | 是，统一脱敏 |
| 编辑弹窗   | 否，展示完整原始值（或根据权限控制） |

***

### 4.3 脱敏存储

推荐使用**国密算法**（如SM4）进行数据加密和脱敏处理。

## 五、文件操作安全

### 5.1 导出安全

| 规则项    | 规则说明                                           |
| ------ | ---------------------------------------------- |
| 单次导出上限 | 单次导出数据量不超过 **1000条**；超出时提示"数据量过大，请缩小筛选范围后重新导出" |
| 操作日志   | 导出行为须记录操作日志                                    |

### 5.2 导入安全

| 规则项    | 规则说明                                     |
| ------ | ---------------------------------------- |
| 文件格式限制 | 仅支持 `.xlsx` 格式，其他格式一律拒绝并提示               |
| 文件大小限制 | 单次上传文件不超过 **10MB**，超出时提示"文件大小不能超过10MB"   |
| 文件条数限制 | 单次上传文件条目不超过 1000 条，超出时提示"文件条目不能超过1000 条" |
| 操作日志   | 批量导入行为须记录操作日志                            |

### 5.3 模板下载安全

| 规则项  | 规则说明                      |
| ---- | ------------------------- |
| 鉴权要求 | 模板下载接口须校验用户登录状态，匿名请求返回401 |
| 模板内容 | 模板文件不含真实业务数据，仅含字段名称和填写示例  |

## 六、数据传输安全

### 6.1 传输加密

| 规则项      | 规则说明                           |
| -------- | ------------------------------ |
| 协议要求     | 全站强制使用 HTTPS，禁止 HTTP 明文传输      |
| HTTP 重定向 | 若用户访问 HTTP 地址，服务端须自动重定向至 HTTPS |

### 6.2 接口鉴权

| 规则项       | 规则说明                                  |
| --------- | ------------------------------------- |
| Token 机制  | 所有需要认证的接口统一使用 Token 鉴权（JWT 或等效机制）     |
| Token 有效期 | Token 有效期与会话超时保持一致（如30分钟无操作失效）         |
| 匿名接口      | 仅登录接口、图形验证码接口、公开页面接口允许匿名访问，其余全部需要鉴权 |
| 越权访问      | 接口层须校验当前用户角色是否有权访问该接口，越权访问返回403       |

### 6.3 接口防护

| 规则项    | 规则说明                                   |
| ------ | -------------------------------------- |
| 登录接口限流 | 同一IP登录接口请求频率限制，防止暴力枚举（建议：同IP每分钟不超过10次） |
| 输入过滤   | 所有接口入参须做过滤和转义，防止SQL注入和XSS攻击            |
| 敏感参数加密 | 密码字段传输前须前端加密（推荐使用SM3），禁止明文传输           |
